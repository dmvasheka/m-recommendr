## Запланированные улучшения

### 9. 🚨 КРИТИЧНО: Оптимизация импорта через очереди

**Проблема:** Фоновый импорт работает неправильно и приводит к исчерпанию лимитов

**Текущие баги:**
1. **Циклический импорт дубликатов**
   - Импорт по кругу добавляет одни и те же фильмы/сериалы
   - Импортируются только первые 40-100 элементов каждой категории (popular, top_rated и т.д.)
   - После первого запроса начинают повторяться и скипаться
   - TMDB возвращает те же страницы, но код не переходит к новым категориям/годам/страницам

2. **Слишком высокая частота запросов**
   - За 2 дня был исчерпан бесплатный лимит Upstash Redis
   - Cron job запускается слишком часто
   - Нет throttling и rate limiting между запросами

3. **Проблема локализации контента**
   - Все фильмы и сериалы импортируются на английском языке
   - Интерфейс русифицирован, но данные (описания, названия) остаются на EN
   - TMDB API поддерживает параметр `language=ru-RU`, но не используется

**Решение:**

#### 9.1 Исправить логику импорта
- [ ] **Отслеживание прогресса импорта**
  - [ ] Сохранять в Redis: `import:movies:lastPage`, `import:tv:lastPage` для каждой категории
  - [ ] Или создать таблицу `import_progress` в БД
  - [ ] Записывать: категория, последняя импортированная страница, timestamp
- [ ] **Умный импорт с прогрессией**
  - [ ] Начинать со страницы N+1, где N - последняя успешно импортированная
  - [ ] Если страница не содержит новых элементов (все скипнуты), переходить к следующей категории
  - [ ] Импортировать из разных годов (1990-2024) циклически
  - [ ] Ротация категорий: popular → top_rated → now_playing → upcoming
- [ ] **Предотвращение дубликатов**
  - [ ] Проверять наличие фильма в БД перед добавлением в очередь
  - [ ] Использовать UNIQUE constraint на `tmdb_id` или `id`
  - [ ] Логировать количество новых vs скипнутых

#### 9.2 Настроить частоту и throttling
- [ ] **Уменьшить частоту cron jobs**
  - [ ] Импорт фильмов: раз в 6-12 часов (вместо ежечасного)
  - [ ] Импорт сериалов: раз в день
  - [ ] Генерация embeddings: по мере накопления (batch раз в час)
- [ ] **Rate limiting для TMDB API**
  - [ ] Задержка между запросами: минимум 250ms
  - [ ] Batch размер: не больше 20 фильмов за раз
  - [ ] Использовать `p-queue` или встроенный BullMQ rate limiter
- [ ] **Upstash Redis оптимизация**
  - [ ] Уменьшить TTL для кэша (если используется)
  - [ ] Минимизировать количество записей в Redis
  - [ ] Рассмотреть переход на локальный Redis для dev/staging

#### 9.3 Локализация импорта
- [ ] **TMDB API с language параметром**
  - [ ] Добавить `language=ru-RU` во все запросы к TMDB
  - [ ] Fallback на EN если русский перевод отсутствует
  - [ ] Сохранять оба языка: `title_en`, `title_ru`, `description_en`, `description_ru`
- [ ] **Мультиязычная схема БД** (опционально)
  - [ ] Таблица `movie_translations` с переводами
  - [ ] Или JSON поле `translations: { en: {...}, ru: {...} }`
- [ ] **API response по языку пользователя**
  - [ ] Frontend передает язык в заголовке или query параметре
  - [ ] Backend возвращает данные на нужном языке
  - [ ] Embeddings генерировать на английском (для консистентности поиска)

#### 9.4 Мониторинг и алерты
- [ ] **Dashboard для импорта**
  - [ ] Количество импортированных фильмов/сериалов за последние 24ч
  - [ ] Статистика скипов (дубликаты)
  - [ ] Использование Upstash Redis (кол-во команд, память)
- [ ] **Логирование**
  - [ ] Подробные логи импорта с категорией, страницей, результатами
  - [ ] Алерты при высокой частоте скипов (>80%)
  - [ ] Предупреждения при приближении к лимиту Upstash

**Порядок работы:**
